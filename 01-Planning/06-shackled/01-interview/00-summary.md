# Inquisition Summary

## 요구사항 요약
- papillon Phase 4(구현)를 독립 스킬 "shackled"로 분리
- 4-A(코드 작업)와 4-B(인프라/절차 작업)를 하나의 스킬로 통합 — 프로세스 골격 공유, 산출물 유형 분기
- 페어 프로그래밍 스킬: 설계서/요구사항 기반으로 AI가 구현하고 사람이 방향을 잡는 협업 도구
- `/papillon:shackled`로 단독 실행 가능, 오케스트레이터 경유도 가능

## 핵심 제약
- 설계서 또는 요구사항 정의서 필수 입력 — 없으면 진행 거부
- S/M 체급만 진입 (L/XL은 하위 태스크로 분해되어 진입)
- 시블링 구조 — shackled은 구현만, wtth 호출은 오케스트레이터 책임
- 태스크 하나씩 순차 구현 (한번에 전부 금지)

## 암시적 전제 조건
- 설계서가 inquisition + wtth 리뷰를 거쳐 확정된 상태에서 진입하는 것이 정상 흐름
- 코딩 전문가 에이전트 활용 필수 (기본 모드에서도)
- 진행 불가 시 롤백 지점은 사람이 결정 — AI는 원인 힌트만 제공
