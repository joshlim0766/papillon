# 5-3: inquisition 호출 인터페이스 미정의

**심각도:** 중간
**분류:** 누락
**상태:** Closed

## 문제

wtth 호출 인터페이스는 파라미터 테이블로 정의되어 있지만, inquisition 호출 시 오케스트레이터가 전달하는 정보(사용자 요구사항 원문, CWD, 기존 컨텍스트 경로 등)가 정의되어 있지 않다.

## 논의

- inquisition은 shackled/wtth과 달리 오케스트레이터가 파라미터를 넘기는 구조가 아님
- 사람과 직접 대화하면서 입력을 수집하는 스킬 — CWD 스캔, 재개 판단도 자체 수행
- 오케스트레이터는 호출만 하고, 반환값(00-interview/)을 받아 다음 스텝으로 진행
- 입력 없이 호출 시 자연스러운 시작 응답("어떤걸 도와드리면 될까요?" 등)은 inquisition 설계서 영역

## 처리

papillon 설계서 Step 1에 "오케스트레이터는 별도 파라미터를 전달하지 않는다. 입력 수집은 inquisition이 사람과 직접 수행한다" 명시.

## 결과

papillon 설계서 §3.1 Step 1에 호출 방식 명시 완료.
