# 3: PRD "사람은 의사결정만" vs 설계의 사람 확인 지점 과다

**심각도:** 높음
**분류:** 모순
**상태:** Open

## 문제

PRD G3이 "사람이 하는 행위는 리뷰 의사결정과 종결 승인뿐"이라고 했는데, 실제 설계는 이 목표를 달성하지 못했다.

★ 마크로 표시된 사람 개입 지점이 M 체급 기준으로도 최소 10곳 이상:
- 인터뷰 제안 승인
- 문맥 파일 생성 승인
- 종료 승인
- 미결 처리
- 체급 승인
- 초안 확인
- finding별 수용/기각 (매 건)
- ADR 승격 승인
- 종결 승인
- 구현 태스크별 확인
- 코드 리뷰 diff별 승인/반려
- 커밋 승인

## 왜 문제인가

"자동화된 파이프라인" 위에 "매 단계 사람 확인"이 덧붙여져서, 수동 파이프라인보다 사람의 개입 빈도가 높아질 수 있다. 각각의 확인 지점에는 이유가 있지만, 전체적으로 보면 "파이프라인은 자동, 사람은 의사결정만"이라는 핵심 목표와 모순된다.

PRD의 약속과 설계의 현실이 다르다는 걸 인식하고, 둘 중 하나를 조정해야 한다:
- PRD를 현실에 맞게 수정하거나
- 설계에서 사람 개입 지점을 줄이거나

## 제안된 솔루션: 확인 지점 3등급 분류

모든 ★를 동등하게 취급하지 말고 3등급으로 분류:

| 등급 | 설명 | 예시 | 동작 |
|---|---|---|---|
| **Gate (관문)** | 잘못 판단하면 이후 전체가 틀어짐 | 체급 판정, 설계 리뷰 종결, P0 finding 수용/기각 | 반드시 사람이 명시적 승인 |
| **Checkpoint (확인)** | 사람이 "OK" 또는 "잠깐" | 문맥 파일 생성, 인터뷰 종료 제안, 태스크 목록 확인 | "이의 없으면 진행합니다" 패턴 가능 |
| **FYI (알림)** | 봐야 하지만 승인 불필요 | 진행률 표시, P2/P3 일괄 처리 결과, 크로스 체크 이력 | 보여주되 멈추지 않음 |

이렇게 분류하면 파이프라인이 사람을 기다리며 멈추는 횟수가 Gate 수준으로 줄어든다. PRD G3의 약속에 훨씬 가까워짐.
