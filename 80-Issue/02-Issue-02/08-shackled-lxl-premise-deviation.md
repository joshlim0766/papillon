# 8: shackled §3.1 "상위 전제 이탈 감지" L/XL 체급 검증 미완

**심각도:** 낮음
**분류:** 공백
**상태:** Open — L/XL 파일럿 후

## 문제

shackled 설계서 §3.1의 "상위 전제 이탈 감지"는 L/XL 태스크에 적용되는 guard인데, 2026-04-14 파일럿은 M 체급 단일 태스크였기 때문에 해당 가드가 실행 대상이 아니었다.

결과적으로 다음이 불명확한 상태로 남아있다:

- 경미한 미스매치(예: 인터페이스 시그니처 1개 차이)를 잡아내는가?
- 중대한 이탈(예: 설계서가 전제한 모듈 구조와 실제 구현 구조 불일치)을 잡아내는가?
- False positive 빈도는 얼마인가?

현 상태에서는 §3.1이 "있으니까 있다" 수준의 미검증 guard다.

## 제안된 솔루션: L/XL 파일럿에서 의도적 미스매치 fixture로 실측

L/XL 체급 파일럿을 기획할 때 다음 3가지 fixture를 포함:

1. **경미한 미스매치**: 상위 전제에서 인터페이스 시그니처 1개만 수정된 케이스 — 감지되는지
2. **중대한 이탈**: 상위 전제에서 가정한 모듈 경계를 하위 태스크가 넘어서는 케이스 — 감지되는지
3. **정상 케이스**: 이탈 없음 — false positive 나지 않는지

측정 항목:
- 감지율 (recall)
- False positive 빈도
- 감지 시점 (태스크 진행 중 어느 단계에서 잡히는가)

**적용 시점:** 다음 L/XL 파일럿 실행 시. 단독으로 이 가드 검증만을 위한 파일럿은 과하고, 실제 L/XL 작업 기회가 생길 때 fixture를 끼워넣는 방식이 효율적.
