# 04-구현 단위 및 진행 불가 처리

**상태:** Settled

## 확인된 사항
- 구현 단위: 태스크 하나씩 순차 (구현 → 크로스 체크 → 사람 확인 → 다음 태스크)
  - 전체 태스크 완료 후 인테그레이션 → 최종 확인 → 산출물 반환
  - 한번에 전부 구현하지 않음
- 진행 불가 처리:
  - shackled이 막히면 구현을 멈추고 사람에게 보고 (왜 막혔는지 + 원인 추적 힌트: "설계서 §X 부분에서 기인한 것으로 보입니다")
  - 사람이 판단하여 결정:
    - 경미: shackled 안에서 바로 해결
    - 중간: 설계 해당 섹션 수정 → 설계 리뷰 재진입
    - 심각: inquisition으로 롤백
  - 롤백 지점은 반드시 사람이 결정. AI는 원인 힌트만 제공
  - shackled은 현재 진행 상태를 저장하고 종료
