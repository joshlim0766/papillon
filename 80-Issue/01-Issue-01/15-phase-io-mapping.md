## 문제

각 Phase가 어떤 산출물을 생성하고, 다음 Phase가 어떤 산출물을 입력으로 받는지 명시적 매핑이 없다. 예시:

- Phase 1 → Phase 2: 설계 초안 파일 경로? `00-interview/` 경로?
- Phase 2 → Phase 3: RDR 경로? 미해결 finding 목록?
- Phase 3 → Phase 4: 확정된 설계서 경로?

흐름은 텍스트로 이해 가능하지만 구현 시 "무엇을 어디서 읽어 어디로 전달하는가"가 모호.
