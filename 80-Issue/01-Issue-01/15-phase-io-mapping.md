# 5-4: Phase 간 산출물 전달(입력/출력) 매핑 미정의

**심각도:** 중간
**분류:** 누락
**상태:** Closed

## 문제

각 Phase가 어떤 산출물을 생성하고, 다음 Phase가 어떤 산출물을 입력으로 받는지 명시적 매핑이 없다. 예시:

- Phase 1 → Phase 2: 설계 초안 파일 경로? `00-interview/` 경로?
- Phase 2 → Phase 3: RDR 경로? 미해결 finding 목록?
- Phase 3 → Phase 4: 확정된 설계서 경로?

흐름은 텍스트로 이해 가능하지만 구현 시 "무엇을 어디서 읽어 어디로 전달하는가"가 모호.

## 논의

- AI 관점에서는 각 Phase 상세에 이미 텍스트로 존재하므로 중복
- 사람이 설계서 읽을 때 전체 흐름을 한눈에 잡는 용도로는 유용

## 처리

§3 도입부에 Phase 간 입출력 매핑 테이블 추가 (Phase 1→2, 1→4(S), 2→3, 3→4, 4→5, 5→6).

## 결과

papillon 설계서 §3 상단에 매핑 테이블 반영 완료.
