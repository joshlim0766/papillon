# 01-구현 모드 및 협업 방식

**상태:** Settled

## 확인된 사항
- 기본 모드 (자동): AI가 설계서 기반으로 코드 작성 → 자체 리뷰(방향 점검) → 결과물을 사람이 확인 (승인/거부/피드백)
- 페어 모드 (협업): 코딩 전문가가 별도 pane에서 AS-IS/TO-BE + 방향 설명 → 사람이 실시간 피드백 → 반복
  - tmux + Claude Code 멀티 인스턴스 실험 플래그 필요
  - JetBrains MCP 경유 시 비동기 방식 (파일 자동 열기까지, 포커스 전환은 불가)
- MVP에서는 기본 모드만 구현, 페어 모드는 설계서에 확장 포인트로 남김
- 입력: 설계서(M) 또는 요구사항 정의서(S) 필수. 없으면 진행 거부
- S/M 체급만 shackled에 진입 (L/XL은 하위 태스크가 S/M으로 분해되어 진입)
