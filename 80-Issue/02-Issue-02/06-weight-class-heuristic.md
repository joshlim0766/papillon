# 6: 체급 판정 경계 heuristic의 경험적 데이터 환류 부재

**심각도:** 낮음
**분류:** 공백
**상태:** Open

## 문제

S/M 경계에 대한 구체적 heuristic이 없다.

shackled 리뷰에서 7개 관심사에 37건 finding이 나온 M 체급 사례가 있는데, 이런 경험적 데이터를 체급 판정 기준에 환류시키지 않았다.

체급 판정 기준 테이블(inquisition §2.7)은 추상적인 기준("단일 관심사 내 설계 판단 필요")만 있고, 실제 사례에서 도출된 구체적 지표(관심사 수, finding 건수 범위 등)가 반영되지 않음.

## 제안된 솔루션: 경험적 캘리브레이션 데이터 축적

체급 기준을 미리 완벽하게 정의하는 건 불가능. 실제 돌려본 사례를 캘리브레이션 데이터로 축적하는 게 현실적.

**기록 형식 예시:**
> M으로 판정했으나 실제로는 L에 가까웠던 사례: 관심사 7개, 1라운드 P0 4건, finding 총 37건 (shackled)

common-spec이나 inquisition 설계서에 "캘리브레이션 사례" 섹션을 두고 축적. 3~5개 사례가 쌓이면 경계가 자연스럽게 보정된다. 지금은 사례가 1개니까 모호한 게 당연.

**적용 시점:** 파이프라인 1~2회 더 돌린 후. 여기에 시간을 쏟기보다 데이터를 모으는 게 효율적.
