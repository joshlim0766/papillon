## 요구사항 요약

### R-1. 핸드오프 브리프 생성 (출처: AI-5)

inquisition이 체급 판정 후 **핸드오프 브리프**를 생성하여 `00-interview/`에 파일로 저장한다. 오케스트레이터가 이를 읽어 Sonnet(초안 작성자)에게 전달한다.

브리프 포함 내용:
1. **맥락 기반 주의 사항** — 인터뷰에서 파악한 설계 시 주의점 (예: "상태 전이가 많으니 status enum 빠짐없이 정의")
2. **깊이 부족 플래그** — 인터뷰에서 충분히 다루지 못한 관심사 표시 (예: "크로스 체크: 언급됨, 상세 미정의. Sonnet 자체 설계 필요")
3. **신뢰도 플래그** — 인터뷰이 답변 일관성 평가
4. **관심사 분포 정보** — 관심사 수, 각 관심사의 상대적 깊이 (AI-7 개입 지점 1의 inquisition 측 기여분)

### R-2. 셀프 오딧 항목 추가 (출처: AI-5)

현행 셀프 오딧(§2.6 항목 13)에 **"관심사별 깊이 균형 확인"** 항목을 추가한다. 이것이 핸드오프 브리프의 깊이 부족 플래그 생성 근거가 된다.

### R-3. 핸드오프 브리프 생성 시점 및 조건

- **시점:** 셀프 오딧 → 스팟 체크 → 사람 최종 확인 → 체급 판정 → **핸드오프 브리프 생성**
- **조건:** S 체급은 브리프 불필요 (설계 초안 없이 바로 구현). M 이상만 생성
- 체급을 알아야 브리프 내용이 달라지므로 체급 판정 이후가 자연스러움

## 핵심 제약

1. **inquisition의 역할 경계 유지** — 브리프는 "이런 점을 주의하라"는 정보 제공이지, 설계 판단이나 솔루션 제안이 아니다
2. **파일 기반 전달** — inquisition → 파일(`00-interview/`) → 오케스트레이터 → Sonnet. 직접 전달 경로 없음
3. **기존 프로세스 최소 변경** — 새 Phase가 아니라 기존 §2.7 체급 판정 뒤에 단계 하나 추가

## 암시적 전제 조건

1. 오케스트레이터(papillon)가 브리프 파일을 인식하고 Sonnet에게 전달하는 로직이 필요 → **papillon 설계서 Step 3 수정 필요** (본 작업 범위 밖, 의존성)
2. papillon Phase 전환 테이블에 브리프가 출력 항목으로 추가되어야 함 → **동일 의존성**
3. AI-7 개입 지점 1(초안 완료 시 관심사 분포 요약)은 오케스트레이터/Sonnet 책임 → **inquisition 범위 밖** 확인됨. 단, inquisition이 브리프에 관심사 분포 정보를 포함하면 오케스트레이터가 이를 활용 가능
