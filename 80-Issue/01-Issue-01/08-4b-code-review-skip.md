# 2-2: 4-B에 코드 변경 포함 시 코드 리뷰 스킵 의도 미명시

**심각도:** 중간
**분류:** 논리적 공백
**상태:** Closed

## 문제

다이어그램에서 Phase 5는 "4-A인 경우"에만 자동 전환된다. 4-B는 Phase 6으로 직행하거나 파이프라인 종료. 그러나 4-B에서도 설정 파일 등 코드 변경이 포함될 수 있다. 이 경우 코드 리뷰 없이 Phase 6으로 가는 것이 의도된 동작인지 명시 필요.

## 논의

CDK, Terraform 등 IaC는 사실상 코드이므로 코드 리뷰가 필요. 4-B 완료 후 `git diff` 대상이 있으면 Phase 5를 경유하도록 분기 추가.

## 처리

4-B 스텝 5의 분기를 3갈래로 확장:
- 코드 변경 포함 (IaC, 설정 파일 등) → Phase 5(코드 리뷰)로 전환
- 커밋 대상 있으나 코드 변경 아님 (런북 문서, RDR 등) → Phase 6
- 커밋 대상 없음 → RDR 생성 후 종료
