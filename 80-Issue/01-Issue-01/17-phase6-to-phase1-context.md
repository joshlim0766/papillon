# 2-5: Phase 6→Phase 1 복귀 시 컨텍스트 전달 미정의

**심각도:** 낮음
**분류:** 논리적 공백
**상태:** Closed

## 문제

Phase 6에서 "다음 태스크 → Phase 1로 복귀"할 때, 같은 세션에서 복귀하는 경우 이전 태스크의 RDR/ADR이 컨텍스트로 누적되는지, 깨끗한 상태에서 재시작하는지 미명시.

## 논의

- 같은 세션에서 다음 태스크로 가면 이전 컨텍스트가 윈도우를 점유 → 품질 저하 위험
- 필요한 맥락은 산출물(RDR/ADR/설계서/index)에 이미 기록되어 있으므로 새 세션의 컨텍스트 스캔으로 추출 가능

## 처리

Phase 6 "다음 태스크" 분기에서 새 세션 시작을 권장하도록 명시.

## 결과

papillon 설계서 §3.6 Phase 6에 새 세션 권장 반영 완료.
