# 01-복합 모드 적용 가이드

**상태:** Settled

## 확인된 사항
- "워크플로우 리뷰"는 정식 모드가 아니다. shackled 리뷰 실험에서 붙인 임시 명칭
- 결론: 별도 모드 신설 불필요. 기존 설계 리뷰 + 운영 절차 리뷰를 복합 적용하고 전문가 풀을 커스텀
- 복합 적용 판별: 오케스트레이터가 시그널 기반으로 제안 → 사람이 최종 확정. 자동 판별이 아닌 제안+확인 방식
- 시그널 예시: 프로세스 단계와 구조 설계가 한 문서에 공존 (shackled 설계서처럼 §2 입력 인터페이스 = 설계, §3 프로세스 흐름 = 절차가 교차)
- 정밀한 자동 판별 기준을 세우기 어려움 — 설계와 절차의 경계가 모호한 문서 유형이라 사람 판단이 필수
- §1.1 모드 선택 규칙에 복합 적용 경로를 추가해야 함
