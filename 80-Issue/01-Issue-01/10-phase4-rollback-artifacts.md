# 2-4: Phase 4 롤백 시 기존 산출물 처리 미정의

**심각도:** 중간
**분류:** 논리적 공백
**상태:** Closed

## 문제

Phase 4 진행 불가 시 세 가지 수준(경미/중간/심각)이 있다. "중간: 설계 해당 섹션 수정 → 설계 리뷰 재진입"과 "심각: inquisition으로 롤백"의 경우, 이미 Phase 4까지 진행된 상태에서 기존 산출물(부분 구현 코드, RDR 등)을 어떻게 처리하는지 미정의.

## 논의

- 설계서, RDR/ADR, 코드 모두 git으로 관리될 확률이 극히 높음
- 산출물 삭제보다 보존이 맞음 — 왜 롤백했는지의 맥락이 이력에 남아야 함
- 사람이 git 이력 + RDR/ADR을 보고 맥락 파악 후 다음 행동 결정

## 처리

Phase 4 공통 골격의 "진행 불가 시" 항목에 롤백 시 산출물 처리 원칙 1줄 추가: 기존 산출물은 삭제하지 않고 git 이력으로 보존, 사람이 맥락 파악 후 판단.

## 결과

papillon 설계서 §3.4 공통 골격에 롤백 산출물 처리 원칙 반영 완료.
