# 1-3: shackled 네임스페이스 호출 방식 미기재

**심각도:** 낮음
**분류:** 구조적 일관성
**상태:** Closed

## 문제

다이어그램에서 "오케스트레이터 → shackled 호출"이라고만 기술. shackled 인터뷰 summary에서는 `/papillon:shackled`로 단독 실행 가능하다고 되어 있으므로, 네임스페이스 구조에 shackled 추가 필요.

## 논의

이슈 3-3(common-spec 네임스페이스에 shackled 누락) 처리 시 함께 해결됨.

## 처리

3-3에서 common-spec §9 설치 구조 + 호출 방식 테이블에 shackled 추가 완료.

## 결과

3-3 처리와 함께 해결 완료.
