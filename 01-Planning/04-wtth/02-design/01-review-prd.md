## 4. 모드 특화 사항

### 4.1. 리뷰 초점

PRD 리뷰는 "구현 가능한 명세인가"를 검증한다. 전문가별 초점:

| 전문가 | 초점 |
|---|---|
| PM | 목표-측정기준 대응, 스코프 경계, 비목표 명확성 |
| ARCH | 기술적 실현 가능성, 제약 조건 충분성, 산출물 구조 |
| SEC | 보안 관련 요구사항 누락, 데이터 흐름 상의 위협 |

### 4.2. 공통 프로세스와의 차이점

#### 스킵 단계
- **ADR 승격 없음**: PRD 리뷰는 아키텍처 결정이 아닌 요구사항 검증이므로, 코어 §3.2 ADR 승격 규칙을 적용하지 않는다.
- **수정 경로 리뷰 해당 없음**: PRD는 papillon Step 3-B(기존 설계서 수정) 대상이 아니므로 코어 §2.3을 적용하지 않는다.

#### 변형 단계
- **전문가 풀 고정 3인**: 설계 리뷰의 6인 풀에서 선별하는 방식과 달리, PRD 리뷰는 PM + ARCH + SEC 3인이 기본이다. 추천 상한 3인(코어 §1.3)과 동일하므로 선별 단계 자체가 생략된다. 선택적 투입(BE, SRE)은 3인 초과이므로 사람 확인이 필요하다.

#### 추가 규칙
- **목표-측정기준 대응 필수 확인**: PM이 "목표마다 측정 가능한 기준이 있는가"를 P0 수준으로 확인한다. 설계 리뷰의 ARCH가 기술적 실현 가능성을 보는 것과 대조적이다.

<rationale>
PRD 리뷰의 전문가 풀을 3인 고정으로 설계한 이유: PRD 단계에서는 기술적 깊이보다 전략적 방향(PM), 실현 가능성(ARCH), 보안 위험(SEC)의 3축이 핵심이다. 도메인 전문가(BE, FE 등)는 PRD 수준에서는 관점이 겹치므로 기본 풀에 넣지 않았다.
</rationale>
