# 02: 수렴 판정 조건이 반영 검증을 포함하지 않음

**심각도:** 높음
**분류:** 공백
**상태:** Open

## 문제

wtth 수렴 전환 조건이 "신규 P0/P1이 0건"만으로 판정한다. 이 조건은 **새로운 지적이 없다**는 것만 확인하고, **이전 라운드에서 수용된 사항이 실제로 반영됐는지**는 검증하지 않는다.

AI가 이전 라운드의 수용 건이 반영되지 않은 상태에서도, 신규 P0/P1만 0건이면 수렴으로 전환될 수 있는 구멍이 존재한다.

## 왜 문제인가

- 수용했지만 반영되지 않은 finding이 수렴/사인오프를 통과하여 리뷰가 종료될 수 있음
- RDR에는 "수용"으로 기록되지만 실제 설계서에는 미반영 — 산출물 간 정합성 깨짐
- Issue-03-01(라운드 간 통합 체크)에서 "수정 반영 단계"를 추가했지만, 이는 다음 라운드 시작 전 체크이지 수렴 판정 조건 자체는 아님

## Issue-03-01과의 관계

- **Issue-03-01**: 라운드 간 "수정이 반영됐는지 체크하는 단계"가 없다 → 해결: §2.1에 수정 반영 단계 5 삽입
- **Issue-03-02 (본 건)**: 수렴 판정 조건 자체가 반영 검증을 포함하지 않는다 → 상위 문제

03-01이 해결되어도, 수렴 전환 조건(`§4.1 신규 P0/P1 0건`)에 "수용 건 반영 완료" 조건이 추가되지 않으면 구멍은 남아 있다.

## 제안된 솔루션

수렴 전환 조건을 강화:

**현재:**
> 직전 라운드 대비 신규 P0/P1이 0건 → 수렴으로 전환

**제안:**
> 직전 라운드 대비 신규 P0/P1이 0건 **+ 이전 라운드 수용 건 전건 반영 완료** → 수렴으로 전환

RDR의 finding 결정 목록에서 "수용" 건의 반영 상태를 추적하고, 미반영 건이 있으면 수렴 전환을 차단한다.

**구체적 구현:** [Issue-05 AI-2 — Sign-off ID Cross-Referencing](../05-Issue-05/02-signoff-cross-referencing.md) 참조. 사인오프 에이전트가 P0/P1 수용 건별로 반영 지점을 인용하고, MCP 문자열 일치로 물리적 검증.
